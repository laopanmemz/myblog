接下来是《“简介”》：

​ 微软今日正式向开发者测试和预览通道发布了 Windows 10 21H1 版本 19043.844 (KB4601382)版本 。参与Windows 10 测试的用户现在可以转到 “设置 >更新和安全” 和 “Windows 更新”，然后选择下载并安装 21H1 更新。

　　据介绍，该版本更新带来了少量改进，包括 Windows Hello 的新功能等。

　　目前新版本测试者较少，具体新功能不详，以下为微软官方更新公告。

　　Windows Hello 多摄像头支持，允许用户在使用带集成摄像头的高端显示器时也可以选择使用外部摄像机。

　　Windows Defender 程序应用程序保护性能改进，包括优化文档打开方案时间。

　　当您打开 Microsoft Defender 应用程序保护 (WDAG) Office 文档时，可能会遇到延迟一分钟多钟的问题，现已修复。微软表示当用户尝试使用通用命名约定(UNC)路径或服务器消息块(SMB)共享链接打开文件时会发生这种情况。

　　提高了用户在复制体积超过 400 MB 的文件时的 Robocopy 性能。

　　修复了导致 WDAG 容器在容器空闲时占用近 1 GB 内存(工作集)的问题。

　　Windows 管理规范(WMI)组策略服务(GPSVC)提供了性能改进，并支持远程工作方案。

　　修复了一个导致 Active Directory(AD)管理员对用户或计算机组成员关系所做的更改传播缓慢的问题。尽管 access token 会更新，但是当管理员使用 gpresult/r 或 gpresult/h 创建报告时，这部分更改不会出现。

　　此外，该版本还包括以下 20H2 改进：

　　我们修复了 Internet Explorer 11 内存泄漏的问题，该问题发生在您使用中文包时。

　　我们修复了某些 COM+ 明号定明策略的问题，该策略导致某些应用程序中死锁。

　　我们修复了一个问题，该问题阻止某些 Win32 应用在使用 runas 时以其他用户方式打开

　　我们修复了在 Windows 开箱即用体验 (OOBE) 期间显示意外屏幕的问题。

　　当 COM 服务器并行将事件传递到多个订阅者时，我们修复了可能导致死锁的问题。

　　我们修复了高级显示设置中显示高动态范围 (HDR) 显示的刷新率不正确的问题。

　　我们修复了一个问题，如果某些 CAD 应用程序依赖于 OpenGL，则可能会阻止这些应用程序打开。

　　我们修复了某些支持低延迟的监视器渲染时可能导致视频播放闪烁的问题。

　　我们修复了有时阻止字符串输入输入方法编辑器 (IME) 的问题。

　　我们修复了由于桌面 Windows 管理器 (DWM) 在远程桌面会话中泄漏句柄和虚拟内存而耗尽资源的问题。

　　我们修复了启动时发生的停止错误问题。

　　当您打开”设置->帐户->登录选项”页面时，我们修复了可能会延迟 Windows Hello 业务 (WHfB) 证书信任部署的问题。

　　我们修复了有时可能阻止某些键盘键工作的问题，例如主页、Ctrl 或左箭头键。将日语 IME 输入模式设置为假名时，将发生此问题。

　　我们从用户帐户配置文件中删除了以前使用的图片的历史记录。

　　更改系统区域设置后，我们修复了在控制台上显示错误语言的问题。

　　我们修复了导致 Windows 远程管理 (WinRM) 的主机进程在格式化来自 PowerShell 插件的消息时停止工作的问题。

　　我们修复了 Windows 管理检测 (WMI) 服务中的问题，该问题导致每次将安全设置应用于 WMI 命名空间权限时导致堆泄漏。

　　在使用某些硬件配置打开游戏后，我们修复了屏幕渲染的问题。

　　当启用用户体验虚拟化 (UE-V) 时，我们改进了具有漫游设置的应用程序的启动时间。

　　我们修复了受信任的 MIT 领域中的主体无法从活动目录域控制器 (DC) 获取 Kerberos 服务票证的问题。在安装包含 CVE-2020-17049 保护和将 PerfromTicketSignature 配置为 1 或更高的 Windows 更新的设备上将发生这种情况。这些更新发布于 2020 年 11 月 10 日至 2020 年 12 月 8 日之间。如果调用方提交无 PAC 票证授予票证 (TGT) 作为证据票证而不提供证据标志，则票证获取也会失败，错误为”USER_NO_AUTH_DATA_REQUIRED KRB_GENERIC_ERROR”。

　　我们在微软端点防御器中修复了高内存和 CPU 利用率。

　　我们在 Microsoft 365 端点中增强了数据丢失防护和内部风险管理解决方案功能。

　　我们修复了支持 Windows Hello 的外部摄像机的问题。默认情况下，当存在内部 Windows Hello 摄像机时，Windows Hello 现在将使用外部摄像头。

　　当您尝试使用 Microsoft Edge 打开不受信任的网页或打开不受信任的 Microsoft Office 文档时，我们修复了显示错误的问题。错误是”WDAG 报告 + 容器： 错误： 0x80070003，Ext 错误： 0x00000001”。安装 .NET 更新 KB4565627 后出现此问题。

　　我们修复了一个阻止 Wevtutil 解析 XML 文件的问题。

　　当椭圆曲线数字签名算法 (ECDSA) 生成 163 字节而不是 165 字节的无效密钥时，我们修复了无法报告错误的问题。

　　我们添加了对使用基于铬的 Microsoft Edge 作为分配的访问单展台应用的支持。现在，您还可以为单个应用展台自定义分组键序列。有关详细信息，请参阅配置微软 Edge 展台模式。

　　我们修复了用户数据报协议 (UDP) 广播数据包大于最大传输单元 (MTU) 的问题。接收这些数据包的设备丢弃它们，因为校验和无效。

　　我们修复了 WinHTTP AutoProxy 服务不符合代理自动配置 (PAC) 文件上为最大生存时间 (TTL) 设置的值的问题。这可防止缓存的文件动态更新。

　　我们提高了 WinHTTP 网络代理自动发现服务忽略动态主机配置协议 (DHCP) 服务器返回的无效 Web 代理自动发现协议 (WPAD) URL 的能力。

　　我们显示正确的信封介质类型作为通用打印队列的可选输出纸张类型。

　　当打印机使用 Microsoft 互联网打印协议 (IPP) 类驱动程序时，我们结束了打印机的随机纸张尺寸显示。

　　我们启用 Windows 来检索更新的打印机功能，以确保用户具有一组正确的可选打印选项。

　　我们更新了对在某些打印机上具有长边第一进纸方向的打印作业的孔冲和装标签位置的支持。

　　我们修复了可能导致 IKEEXT 服务间歇性停止工作的问题。

　　我们修复了可能阻止非易失性内存快速 (NVMe) 设备进入适当电源状态的问题。

　　我们修复了可能导致运行网络文件系统 (NFS) 服务的服务器上系统中的停止错误 7E 的问题。

　　我们修复了阻止用户配置文件服务可靠地检测慢速或快速链接的问题。

　　使用工作文件夹时，我们修复了导致元数据锁争用的问题。

　　我们添加了一个新的 dfslogkey，如下所述：

　　关键路径：HKEY_LOCAL_MACHINE/软件/微软/dfslog。

　　RootShareAcquire 成功事件字段具有以下可能的值：

　　默认值 = 1;启用日志。

　　1 以外的值;禁用日志。

　　如果此密钥不存在，将自动创建它。为使生效，对注册表中 dfslog/RootShareAcquire 成功事件的任何更改都需要重新启动 DFSN 服务。

　　我们更新了开放移动联盟 (OMA) 设备管理 (DM) 同步协议，将客户端请求的签入原因添加到服务器。签入原因将允许移动设备管理 (MDM) 服务对同步会话做出更好的决策。通过此更改，OMA-DM 服务必须与 Windows OMA-DM 客户端协商 4.0 的协议版本。

　　默认情况下，我们在 Windows Internet (WinINet) 中关闭了令牌绑定。

　　我们修复了一个问题，可能会阻止正确的汉字出现在自动允许输入汉字的应用中。您可能需要手动输入汉字字符。使用 Microsoft 日语输入方法编辑器 (IME) 在这些应用中输入汉字字符时，将发生此问题。
